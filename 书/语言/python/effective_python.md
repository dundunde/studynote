## 编程风格

### 空格

1. 对于占据多行的长表达式来说，除了**首行之外**的其余各行都应该**在通常的缩进级别之上**再加**4个空格**。

2. 在一个文件中，**函数和类**之间用**两个空白行分隔**

3. 在**类**内部，**方法**之间用**一个空白行分隔**

4. 在变量赋值前后，**等号 `=` 的左边和右边，必须且只能各加一个空格**

    ```python
    # ❌错误示范
    x=1
    if x==1:
        pass
    x+=1
    
    # ✅ 正确示范
    x = 1
    if x == 1:
        pass
    x += 1
    ```

    

5. 使用字典时，**冒号左边不留缝，冒号右边空一格**

    ```python
    # ❌错误示范
    d = {
        'name':'lihua',	# 冒号与值之间未加空格
        'age' : 10,		# 两端都加了空格
    }
    
    # ✅ 正确示范
    d = {
        'name': 'lihua',# 冒号与值之间加空格
    }
    
    ```

    

6. 不要将空格放在列表索引、函数调用或关键字参数赋值的两侧

    - 当你使用方括号 `[]` 来访问**列表、元组或字典**的元素时，**括号内外**都不要加空格。

    ```python
    # ❌错误示范
    items [ 3 ]
    data [ 'key' ]
    items [ 1 : 3 ]
    
    # ✅ 正确示范
    items[3]
    data['key']
    items[1:3]
    ```

    - 调用函数时，**函数名与括号之间**、**括号与内容**之间**不要加空格**

    ```python
    # ❌错误示范
    print ( 'Hello' )
    
    # ✅ 正确示范
    print('Hello')
    ```

    - 关键字参数赋值

        这是最容易混淆的一点

        在**普通变量赋值时**，我们需要加空格(例如 `x = 1`)

        但是在函数定义或者函数调用传递参数时，等号`=`两边不要加空格

    ```python
    # ❌ 错误示范
    def func(timeout = 30)
    
    func(timeout = 100)
    
    # ✅ 正确示范
    #等号两边紧凑
    def func(timeout=30)
    
    fun(timeout=100)
    ```

    

7. 给变量做类型注解（annotation）时，空格规则和**字典的键值对规则一样**(第五条)

    ```python
    def greet(name: str):
        pass
    ```

    当类型注解遇上**默认值**，等于号`=`两边各一个空格，参照第4条

    ```python
    # 赋值的等于号两边各一个空格
    def greet(name: str = 'Alice')
    ```

### 命名

- 函数、变量及属性用小写字母来拼写，各单词之间用下划线相连，例
    - 如：lowercase_underscore。
- 受保护的实例属性，用一个下划线开头，例如：
    - _leading_underscore。
- 私有的实例属性，用两个下划线开头，例如：
    - __double_leading_underscore。
- 类（包括异常）命名时，每个单词的首字母均大写，例如：
    - CapitalizedWord。
- 模块级别的常量，所有字母都大写，各单词之间用下划线相连，例
    - 如：ALL_CAPS。

### 表达式和语句

1. 采用行内否定

    > 即把否定词直接写在要否定的内容前面，而不要放在整个表达式的前面，例如应该写if a is not b，而不是if not a is b。

2. 不要使用长度判断容器或序列是否为空

    > 例如不要通过if len(somelist) == 0判断somelist是否为[]或''等空值，而应该使用 if not somelist 来判断，因为Python 会将空值(**空列表、空字典、空字符串、空元组、数字0、空值None**等)返回False，而非控制会返回True

3. 判断容器或序列里面有没有内容，也不应通过长度来判断，而是通过if somelist语句(参考第二条)

4. 如果表达式一行写不下，可以用括号将其括起来，而不要用\符号续行，而且要适当地添加换行与缩进以便于阅读。

    ```python
    add_all = (1
    	+ 2
        + 3
        + 4    
    	)
    ```

    

5. 如果**一行**字符串太长时，可以使用括号 + 引号自动合并 

    这是 Python 的一个“魔法”特性：**只要把两个字符串字面量（String Literals）并排放在一起，Python 就会自动把它们拼成一个。**

    ```python
    my_long_url = (
        "https://www.example.com/api/v1/users/"
        "details/search?q=python"
        "&sort=descending"
    )
    
    info = (
        f"User name is {name}, "
        f"and the age is {age}."
    )
    ```

    

6. 如果字符串长且**需要换行**，可以使用三引号 `"""` 或 `'''`

    ```python
    poem = """
    白日依山尽，
    黄河入海流。
    """
    ```

    ⚠️ 一个常见的“缩进陷阱”

    在函数内部使用三引号时，要非常小心**缩进**。三引号会把**代码左边的缩进空格也当作字符串的一部分！**

    ```python
    def print_menu():
        # 为了代码对齐，你加了缩进，但这些空格会被打印出来
        menu = """
        1. Start
        2. Exit
        """
        print(menu)
    
    # 输出结果（前面会有一大堆难看的空格）：
    #         1. Start
    #         2. Exit
    ```

    **✅ 解决方案（高手用法）：使用 `textwrap.dedent`** 如果想让代码整齐，又不想保留左边的缩进，可以使用标准库 `textwrap`。

    ```python
    from textwrap import dedent
    
    def print_menu():
        # dedent 会自动切除左边公共的缩进空格
        menu = dedent("""
            1. Start
            2. Exit
        """).strip() # .strip() 去掉首尾多余的空行
        print(menu)
    ```

    

### 模块引入

1. 引入模块时，应使用**绝对名称**，而不是根据当前模块路径使用相对名称

    > 例如：
    >
    > 要引入 bar 包中的 foo 模块，应该**完整写出 from bar imort foo**，即便当前在bar包中，也不应简写为 import foo

2. 如果一定要使用相对名称来编写import语句，那就应该明确写成：form . import foo

3. 文件中的import语句应该按顺序划分成三个部分：首先引入标准库里的模块，然后引入第三方模块，最后引入自己的模块。属于同一个部分的import语句**按字母顺序排列**。

with open('data.bin', 'w') as f:

```python
with open('data.bin', 'r') as f:
	data = f.read()
```

如果想要以文本形式读取二进制数据不报错，就得找到适合该二进制的编解码方式(如果有的话)，然后通过encoding进行指定，这样就能将二进制数据转换为文本
